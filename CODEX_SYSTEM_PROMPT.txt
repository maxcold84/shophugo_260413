This repository is static-first and hypermedia-driven.

Non-negotiable architecture:
- Hugo 0.160.1 owns public crawlable storefront pages, SEO output, and static rendering.
- PocketBase 0.36+ is the only long-running application process.
- PocketBase pb_hooks JavaScript only.
- Browser contract is HTML documents and HTML fragments.
- htmx transports HTML fragments.
- Alpine.js is only for small UI state and micro-interactions.
- Do not introduce a browser-facing JSON application API.
- Do not drift into SPA architecture.

Required stack:
- Hugo 0.160.1
- PocketBase 0.36+
- PocketBase pb_hooks JavaScript only
- PocketBase JS migrations
- htmx 2.x
- Alpine.js
- Tailwind CSS v4 through Hugo css.TailwindCSS
- TOAST UI Editor for CMS only, self-hosted

Forbidden:

- CDN frontend assets

- placeholder code

- architecture shortcuts that violate the revised contract docs

Implementation invariants:
- Pages are static. State is dynamic.
- All authoritative money values use integer minor units.
- PocketBase is authoritative for auth, runtime state, validation, fragment rendering, checkout, stock, and build orchestration.
- Hugo is authoritative for SEO-critical storefront rendering.
- localStorage is draft convenience state only and never authoritative.
- Security failures must fail closed.
- Build queue state must be durable and restart-safe.
- No timer-based debounce in pb_hooks.
- No in-memory queue state as durable truth.
- No admin or privileged auth token may be exposed to browser JavaScript.

Phase-one product boundary:
- Keep authenticated cart in phase one.
- Support guest cart, authenticated cart, guest-to-auth cart merge, checkout prepare, checkout submit, and authoritative stock decrement.
- Do not build a full storefront customer account system.
- Do not add account pages, order history UI, profile management UI, or customer account navigation unless explicitly requested later.

Current revised contract changes to enforce:
- `/cms/login` is the only public entry point inside the CMS route namespace; other `/cms/**` routes require CMS auth.
- Checkout success must never exist unless all authoritative stock decrements succeed.
- If full transactional rollback is unavailable, use explicit compensating failure handling and preserve an auditable failed order state.
- Build stale-lock recovery must be evaluated independently of `queue_dirty`.
- Export normalized storefront JSON after lock acquisition and immediately before Hugo invocation.
- Guest cart sync must use POST and HTML form encoding as non-authoritative intent.
- `build_state` is a singleton operational collection.
- `products.tags` is a JSON array of strings with deterministic ordering.
- `cms_sessions.admin_ref` stores the authenticated superuser record id as text.
- robots rules must not block public asset delivery such as `/api/files/`.

PocketBase pb_hooks coding rules:
- conservative ES5-style JavaScript only
- no arrow functions
- no async/await
- no Promise assumptions
- no ESM import/export
- no class syntax
- no browser globals
- use require()
- prefer explicit validation and deterministic flows
- keep handlers simple, synchronous where practical, and restart-safe
